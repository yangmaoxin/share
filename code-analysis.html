B 步骤详解

1

CLI 命令注册与配置

▼

使用 @jackwener/opencli 框架声明一个名为 stats 的命令， 配置目标域名、浏览器策略、参数限制和输出列定义。

TypeScript 
export const statsCommand = cli({
  site: 'channels',
  name: 'stats',
  strategy: Strategy.UI,    // 使用 UI 自动化策略（浏览器操作）
  browser: true,             // 启用浏览器
  domain: 'channels.weixin.qq.com',
  args: [
    { name: 'limit', type: 'int', default: 20 },
  ],
  columns: ['title', 'time', 'views', 'likes', 'comments_count', 'shares', 'thumbs', 'cover'],
  func: async (page, kwargs) => { ... }
});
关键配置作用
Strategy.UI通过浏览器模拟用户操作来抓取数据
browser: true启用 Puppeteer 浏览器实例
args: [{ limit }]接受命令行参数控制最大抓取视频数
columns定义输出的列结构，用于表格化展示

选择器提取内容

▼

跳转到评论管理页面，逐个点击视频展开评论列表，提取用户名、头像、时间和内容。

TypeScript 
for (let i = 0; i < limit; i++) {

4

数据合并（标题匹配关联）

▼

通过视频标题将统计数据和评论数据进行关联，生成完整的最终结果。

TypeScript 
const finalResults = statsList.map((stat) => {
  // 在评论数据中寻找相同标题的视频
  const matchedCommentData = commentsList.find(
    (c) => c.title === stat.title
  );
  return {
    ...stat,
    comments_list: matchedCommentData
      ? matchedCommentData.comments
      : []
  };
});
⚠ 潜在风险： 使用标题字符串匹配，如果标题存在细微差异（空格、截断、特殊字符）会导致匹配失败。建议使用视频唯一 ID 进行关联。
5

写入 MySQL 数据库

▼

连接 MySQL 数据库，创建表结构，使用 UPSERT 策略写入视频和评论数据。

TypeScript 
const connection = await mysql.createConnection({
  host: '10.1.28.16',
  port: 3306,
  user: 'root',
  password: 'root',
  charset: 'utf8mb4'  // 关键！支持 emoji 和复杂汉字
});

// 强制会话字符集
await connection.query('SET NAMES utf8mb4 COLLATE utf8mb4_unicode_ci');
// 修改数据库字符集
await connection.query('ALTER DATABASE ... CHARACTER SET utf8mb4');

utf8mb4 是 MySQL 中唯一能完整存储 4 字节 Unicode 字符（如 emoji）的字符集。 微信昵称和评论中经常包含 emoji，用普通 utf8（3 字节）会导致数据丢失。

SQL 
-- 视频统计表
CREATE TABLE IF NOT EXISTS video_stats (
  id       INT AUTO_INCREMENT PRIMARY KEY,
  title    VARCHAR(255) NOT NULL,
  time     VARCHAR(100),
  views    INT DEFAULT 0,
  likes    INT DEFAULT 0,
  comments_count INT DEFAULT 0,
  shares   INT DEFAULT 0,
  thumbs   INT DEFAULT 0,
  cover    VARCHAR(1024),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_title_time (title, time)
);

-- 评论表
CREATE TABLE IF NOT EXISTS video_comments (
  id       INT AUTO_INCREMENT PRIMARY KEY,
  video_id INT NOT NULL,
  user     VARCHAR(255),
  time     VARCHAR(100),
  content  TEXT,
  avatar   VARCHAR(1024),
  FOREIGN KEY (video_id) REFERENCES video_stats(id)
    ON DELETE CASCADE
);
TypeScript 
// 视频 UPSERT：存在则更新，不存在则插入
await connection.query(`
  INSERT INTO video_stats (title, time, views, likes, ...)
  VALUES (?, ?, ?, ?, ...)
  ON DUPLICATE KEY UPDATE
    views = VALUES(views),
    likes = VALUES(likes), ...
`, [safeTitle, result.time, ...]);

// 评论：先删后插（保证数据最新）
await connection.query(
  'DELETE FROM video_comments WHERE video_id = ?',
  [videoId]
);
await connection.query(`
  INSERT INTO video_comments (video_id, user, time, content, avatar)
  VALUES ?
`, [commentValues]);

D 潜在问题与改进建议

🔴 高风险

数据库凭据硬编码

root/root 直接写在代码中，存在严重安全隐患。应使用环境变量或配置文件管理敏感信息。

🟡 中风险

标题匹配不可靠

通过标题字符串匹配关联数据，如果标题有细微差异（空格、截断）就会匹配失败，建议使用视频唯一 ID。

🟡 中风险

硬编码等待时间

setTimeout 固定等待 4 秒，页面加载慢时会失败。建议使用 waitForSelector 或 waitForNavigation。

🔵 性能

评论逐个点击效率低

每个视频需点击 + 等 2 秒，limit=20 时至少 40 秒。可考虑并行请求或直接调用内部 API。

🟡 中风险

URL 反引号问题

page.goto 中的 URL 使用了反引号包裹，可能是模板字符串转义问题，实际运行时可能导致导航失败。

🔵 优化

评论先删后插策略

每次运行都清空旧评论重新插入，虽然简单但效率不高。大数据量时可考虑增量对比更新。
