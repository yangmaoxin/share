核心抓取逻辑

微信视频号后台数据抓取的关键步骤

1

穿透 iframe 抓取视频统计元数据

微信后台页面内容被深层嵌套在 iframe 中，需要获取 iframe.contentDocument，然后遍历 .post-feed-item 节点，提取标题、时间、封面、播放量、点赞等数据。

2

模拟点击抓取评论详情与头像

跳转到评论管理页，模拟真实用户的鼠标点击左侧视频列表项，等待右侧评论区加载完成后，提取评论人的头像和内容。

3

数据合并与智能入库

通过视频的 title 将两个步骤抓取到的数据进行合并映射，利用 mysql2 库将数据持久化，使用 ON DUPLICATE KEY UPDATE 实现增量更新。

自动回复功能

基于评论关键字的全自动回复实现

1

破解“点击失效”

使用 scrollIntoView() 强行滚动到中心，并用 dispatchEvent(new MouseEvent('click', { bubbles: true })) 向底层 SVG 气泡图标派发最真实的鼠标事件。

2

突破 React 状态锁

使用 JavaScript 原型链获取原生 Setter 注入值，然后立刻派发 input 事件，完美骗过 React 状态树，点亮“发送”按钮。

3

跨 iframe 原型链

精准获取 iframe.contentWindow.HTMLTextAreaElement.prototype，拿到属于该 iframe 上下文的底层 Setter。

if (content.includes("意思")) {
  // 1. 强行滚动并点击底层的回复小图标
  const replyBtn = el.querySelector(".weui-icon-outlined-comment");
  replyBtn.scrollIntoView({ behavior: 'smooth', block: 'center' });
  await wait(500);
  replyBtn.dispatchEvent(new MouseEvent('click', { bubbles: true, cancelable: true, view: iframe.contentWindow }));
  
  // 2. 突破 iframe 原型链与 React 状态锁，强制写入内容
  const textarea = el.querySelector("textarea");
  const nativeInputValueSetter = Object.getOwnPropertyDescriptor(
    iframe.contentWindow.HTMLTextAreaElement.prototype, "value"
  )?.set;
  nativeInputValueSetter.call(textarea, "欢迎观看这个视频谢谢");
  textarea.dispatchEvent(new Event("input", { bubbles: true }));
  
  // 3. 定位并点击“发送”按钮
  const submitBtn = Array.from(el.querySelectorAll(".tag-wrap")).find(b => b.className.includes("primary"));
  if (submitBtn && !submitBtn.className.includes("disabled")) {
    submitBtn.click();
  }
}

跨设备部署指南

通过 OpenCLI 轻松将方案迁移到任何电脑上运行

1

全局安装 OpenCLI

npm install -g @jackwener/opencli

2

安装数据库驱动

cd ~/.opencli && npm install mysql2

3

注册脚本命令

opencli register /路径/到/你的/channels-stats.ts

4

执行抓取与自动回复

opencli channels stats --limit 20
opencli channels auto_reply --limit 20
