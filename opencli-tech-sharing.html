Overview

什么是 OpenCLI？

OpenCLI 是一个开源的浏览器自动化 CLI 框架，核心理念是 "Make Any Website Your CLI" —— 让任何网站操作变成可执行的命令行指令。

🔌

自动注册发现

将 .js 或 .yaml 文件放入 clis/ 目录，OpenCLI 自动发现并注册指令，无需手动配置

🎭

多策略支持

支持 Cookie、UI 自动化、API 拦截、公开接口等多种数据获取策略

🌐

浏览器操控

基于 Playwright，可模拟点击、输入、滚动等浏览器操作，实现复杂交互

📊

结构化输出

支持 table、json、yaml、csv 等多种输出格式，方便与其他工具集成

Step by Step

操作步骤详解

从零开始注册一个自定义指令的完整流程。

定位 OpenCLI 配置目录

OpenCLI 的自定义指令存放在用户目录下的 .opencli/clis/ 文件夹中。每个站点对应一个子目录。

📁 C:\Users\admin\.opencli\
├── 📁 clis/
│ ├── 📁 channels/   ← 微信视频号指令
│ │ ├── comments.js
│ │ └── stats.js
│ ├── 📁 xiaohongshu/   ← 小红书指令
│ │ ├── note.js
│ │ ├── search.js
│ │ ├── feed.yaml
│ │ ├── notifications.yaml
│ │ └── reply-dm.js   ← 我们新增的！
│ └── 📁 ...其他站点/
└── adapter-manifest.json
💡
目录名即为站点名（site），文件名即为指令名（name）。例如 xiaohongshu/reply-dm.js 对应指令 opencli xiaohongshu reply-dm。

编写指令文件

使用 cli() 函数注册指令，定义站点、名称、策略、参数和处理逻辑。核心结构如下：

JavaScript — 指令模板
import { cli, Strategy } from '@jackwener/opencli/registry'; cli({ site: 'xiaohongshu', // 站点标识，对应目录名 name: 'reply-dm', // 指令名，对应文件名 description: '自动回复小红书私信', // 描述，显示在 list 中 domain: 'www.xiaohongshu.com', // 目标域名 strategy: Strategy.UI, // 策略模式 browser: true, // 是否需要浏览器 args: [ // 命令行参数定义 { name: 'message', type: 'string', default: '你好' }, { name: 'limit', type: 'int', default: 10 }, ], columns: ['rank', 'user', 'last_message', 'replied'], func: async (page, kwargs) => { // 核心逻辑 // page: Playwright Page 对象 // kwargs: 用户传入的参数 await page.goto('https://...'); const data = await page.evaluate(`...`); return data; }, });

核心 API 说明

在 func 函数中，你主要通过 page 对象操控浏览器。以下是最常用的 API：

API作用示例
page.goto(url) 导航到指定页面 await page.goto('https://...')
page.wait(seconds) 等待指定秒数 await page.wait(3)
page.evaluate(js) 在浏览器中执行 JS 并返回结果 const data = await page.evaluate(`...`)
page.screenshot() 截图保存（调试用） await page.screenshot({path: '...'})
page.setFileInput() 通过 CDP 上传文件 await page.setFileInput(paths, sel)
💡
page.evaluate() 是最核心的 API —— 它在浏览器上下文中执行 JavaScript 代码，可以访问 DOM、读取页面数据、模拟用户操作，并将结果返回到 Node.js 环境。

解决 TypeScript 加载问题

OpenCLI 基于 Node.js 运行，无法直接加载 .ts 文件。如果目录中同时存在 .ts 和 .js 文件，.ts 文件加载会报错（但 .js 仍可正常加载）。

⚠️ 错误信息
⚠ Failed to load module .../comments.ts: Unknown file extension ".ts" ⚠ Failed to load module .../stats.ts: Unknown file extension ".ts"
⚠️
解决方案：为每个 .ts 文件创建对应的 .js 版本。.js 文件内容与 .ts 基本一致，只需移除 TypeScript 类型注解（如 : string、: int 等）。OpenCLI 会优先加载 .js 文件。

Practice

实战：小红书自动回复私信

完整展示 reply-dm.js 的实现思路与核心代码。

执行流程

导航到私信页
→
检测登录状态
→
扫描会话列表
→
逐个点击会话
→
输入回复内容
→
点击发送
JavaScript
// 导航到私信页面 await page.goto('https://www.xiaohongshu.com/message'); await page.wait(3); // 检测是否需要登录 const loginCheck = await page.evaluate(` (() => { const text = (document.body?.innerText || '').replace(/\\s+/g, ' ').trim(); const needLogin = /登录后查看|请登录|登录小红书/.test(text); return { needLogin }; })() `); if (loginCheck.needLogin) { throw new Error('需要登录才能查看私信'); }
💡
登录检测通过正则匹配页面文本中的关键词实现。这是最简单且通用的方式，适用于大多数需要登录的网站。
JavaScript
// 扫描私信会话列表 const conversations = await page.evaluate(` (() => { const results = []; const items = document.querySelectorAll( '.message-item, .chat-item, [class*="session-item"], [class*="conversation-item"]' ); items.forEach((item, idx) => { const clean = (el) => (el?.textContent || '').replace(/\\s+/g, ' ').trim(); const userName = clean( item.querySelector('[class*="name"], [class*="nickname"]') ); const lastMsg = clean( item.querySelector('[class*="content"], [class*="preview"]') ); if (userName) { results.push({ userName, lastMsg, elementIndex: idx }); } }); return results; })() `);
💡
CSS 选择器使用了多个候选值（逗号分隔），这是因为不同版本的网站 DOM 结构可能不同。这种"模糊匹配"策略提高了指令的兼容性。
JavaScript
// 1. 点击目标会话 await page.evaluate(` (() => { const items = document.querySelectorAll( '.message-item, .chat-item, [class*="session-item"]' ); items[${idx}].click(); // 点击第 N 个会话 })() `); await page.wait(2); // 等待聊天窗口加载 // 2. 在输入框中输入回复内容 await page.evaluate(` (() => { const el = document.querySelector( '[contenteditable="true"][class*="input"], textarea[class*="input"]' ); el.focus(); el.textContent = ''; document.execCommand('insertText', false, '你好'); el.dispatchEvent(new Event('input', { bubbles: true })); })() `); // 3. 点击发送按钮或按 Enter await page.evaluate(` (() => { const btn = document.querySelector('[class*="send"]'); if (btn) btn.click(); })() `);
⚠️
使用 document.execCommand('insertText') 而非直接设置 textContent，因为前者会触发 React/Vue 的输入事件监听，确保框架能感知到输入变化。

YAML 方式：更简洁的指令定义

对于简单的数据抓取（无需复杂交互），可以用 YAML 格式定义指令，无需编写 JS 代码：

YAML — notifications.yaml
site: xiaohongshu name: notifications description: "小红书通知 (mentions/likes/connections)" domain: www.xiaohongshu.com strategy: intercept browser: true args: type: type: str default: mentions limit: type: int default: 20 columns: [rank, user, action, content, note, time] pipeline: - navigate: https://www.xiaohongshu.com/notification - tap: store: notification action: getNotification capture: /you/ select: data.message_list - map: rank: ${{ index + 1 }} user: ${{ item.user_info.nickname }} - limit: ${{ args.limit }}
💡
YAML 模式使用 pipeline 管道语法，通过 navigate → tap → map → limit 四步即可完成数据抓取。适合 Strategy.INTERCEPT 策略，拦截浏览器网络请求获取数据。
Notes

注意事项 & 最佳实践

开发自定义指令时需要牢记的关键要点。

  • !
    必须使用 .js 格式：OpenCLI 无法直接加载 .ts 文件。如果源文件是 TypeScript，需要编译或手动转换为 JavaScript。目录中同时存在 .ts 和 .js 时，.ts 会报错但不影响 .js 加载。
  • !
    CSS 选择器要做兼容：网站 DOM 结构可能随时变化，使用多个候选选择器（逗号分隔）和属性模糊匹配（[class*="name"]）提高兼容性。
  • !
    输入内容用 execCommand：直接设置 textContent 或 value 不会触发 React/Vue 的状态更新。使用 document.execCommand('insertText') 模拟真实输入。
  • ✓
    提供 --dry-run 选项：对于有副作用的操作（发消息、发布内容），务必提供 dry-run 模式，让用户先预览再执行。
  • ✓
    合理的等待时间：页面加载、动画过渡都需要时间。使用 page.wait() 确保元素渲染完成后再操作。
  • !
    不要破坏现有指令：新增文件时不要修改同目录下已有的指令文件。OpenCLI 会自动发现新文件，无需修改任何配置。
  • ✓
    登录检测是必须的：所有需要登录的指令都应该在开始时检测登录状态，给出清晰的错误提示。
  • !
    截图调试：当操作失败时，使用 page.screenshot() 保存截图，方便排查问题。
